Not Weird. Queer

Not Weird. Queer is a loosely autobiographical text-based video game about a trans teen going through middle school.

Currently, Not Weird. Queer is still in the alpha-af (pre-alpha) stage of developement.

Play the latest version: 0.0.006 alpha-af

Alpha-af

Alpha-af versions are work-in-progresses. They do not include the full story and contain dummy dialogue and skeleton pages. Point systems are subject to change without notice.

Report bugs and grammar inconsistancies here.

Version 0.0.006

Version 0.0.006 is a bug fix for the v. 0.0.005 female-route name prompt issue.

  • Known issue: when choosing a name, the Harlowe engine will occasionally display an error message. If you click "OK" on the message, the game will resume normally.
  • Loop logic now depends on $loop, a variable tracking the absolute day as written in the code. Each unique day has a unique $loop number, and $loop numbers repeat when days repeat.
    • If you find any errors in the loop logic, such as skipping days, sequentially repeated days, or events appearing on the wrong days, please submit a bug report.
  • v. 0.0.005 bug fixes:
    • Fixed the bug where the female path is unplayable because the prompt for choosing a name in that path is broken.
  • Known bug: if you input a name that is already the ally's name or your deadname, the Harlowe engine will run into an error.
 

Version 0.0.005

Version 0.0.005 includes a mostly fleshed-out, playable part 2 loop and further improvements to the part 1 loop.

  • Known issue: when choosing a name, the Harlowe engine will occasionally display an error message. If you click "OK" on the message, the game will resume normally.
  • Comprehensive spelling, grammar, and white-space corrections.
    • If you find anymore spelling or grammar issues, please submit a bug report.
  • Expansion to the playable part 2 loop (still incomplete).
  • v. 0.0.004 bug fixes:
    • Fix the bug where if you come out to the ally on the 4th day after day 1 (day 5, day 9, etc), the game will run into an error and send you home instead of taking you to the book club.
    • Fixed numerous instances where the name "Ryan" was hard-coded into the text instead of dynamically showing the name of ally for the current play-through.
  • Known bug: if you input a name that is already the ally's name or your deadname, the Harlowe engine will run into an error.
  • Known bug: the female path is unplayable because the prompt for choosing a name in that path is broken.
 

Version 0.0.004

Version 0.0.004 includes an improved part 1 loop and the start of the part 2 loop.

  • Known issue: when choosing a name, the Harlowe engine will occasionally display an error message. If you click "OK" on the message, the game will resume normally.
  • Now more variety for the ally's name (randomized), player character's deadname (randomized), and player character's default name suggester.
  • Slight part 1 point-based action balance tweaks and changes to dummy text and skeleton pages.
  • v. 0.0.003 bug fixes:
    • Fixed the bug where every 4th day after day 3 (day 7, day 11, etc.), if you choose to talk to the ally during PE after defending your gender, you will skip lunch that day.
  • Known bug: if you input a name that is already the ally's name or your deadname, the Harlowe engine will run into an error.
 

Version 0.0.003

Version 0.0.003 includes a fully playable part 1 loop.

  • Completely overhauled the story loop.
  • Reworked the point-based action balances for hidden variables:
    • Each applicable action only increases or decreases happiness by one point at most.
    • No actions affect self-esteem points - these are calculated at the end of each day and are cumulative.
    • Added ally points that affect the progress of part 1 at certain thresholds.
  • Changed the ally's name from Ryan to a dynamic variable based on the gender of the player character.
  • Smoothed some skeleton text.
  • Known bug: every 4th day after day 3 (day 7, day 11, etc.), if you choose to talk to the ally during PE after defending your gender, you will skip lunch that day.
 

Version 0.0.002

Version 0.0.002 includes a fully playable day 1.

  • Known bug: Players can give the player character the same name as their intended deadname or the ally's name.
  • Known bug: If Ryan doesn't save the player during lunch, storming out of the cafeteria leads to a page with a broken else statement.
